CONTAINERS

A container is a package of all the code and dependencies used to run an application on any computing environment. Containers virtualize the OS of a development environment to run the application. Unlike with virtual machines, containers virtualize the operating system for the application rather than virtualizing the hardware.

When building a container, Docker is running an image that the developer creates of the application. Containers make it easy to run an application reliably in any cloud environment.

IMAGES

An image is a snapshot of an application, its dependencies, and the operating system it runs on. A Dockerfile is a file that contains the code that tells Docker how to build an image. Images are used to build and run containers that run the application and can be deployed on the cloud. Images can be pushed and pulled from a Docker registry such as Docker Hub to easily build containers and run them in any deployment environment.

COMMANDS

docker ps - lists all the running containers
docker kill <container-id> - shuts off the container with the specified <container-id>
docker rm <container-id> - deletes the container with the specified <container-id>
docker compose up - Builds, (re)creates, starts, and attaches to containers for a service
docker compose down - Stops containers and removes containers, networks, volumes, and images created by up
docker build -t <container> . - Builds a container image with the specified <container> image
docker run -dp 80:80 <container> - Runs the container in the background at the specified port numbers (80:80 in this case)
docker commit --change='CMD apachectl -DFOREGROUND' -c "EXPOSE 80" <container-id> talaar/first-website:latest - Makes a commit to a container with a new image, updating either the container's files or settings that were changed. In this case, the --change option adds a command (CMD) to run when running the image.
docker images - Lists all the images
docker push talaar/first-website:latest - Pushes the container to Docker hub
docker pull talaar/first-website:latest - Pulls the image from Docker hub onto your computer
docker login - Allows you to login with your Docker Hub username and password to push to your repository

STARTING THE PROJECT

To start the project, you first need to download Docker.

You can get started with downloading Docker here.

Once Docker is downloaded, open a terminal.
The first thing to do is to pull the image of the project from the Docker Hub repo.
To do this, enter the following command into the terminal: docker pull talaar/is601-flask:master.

Then, run docker run -dp 80:5000 talaar/is601-flask:master.
This will use the project's image to build and run the container.

To see the container running, you can enter the docker ps command into the terminal to see the running container listed.
To see the project running, you can type in localhost into a tab on your browser.

To stop the container, type docker ps into the terminal, copy the container id of the running container, and then enter the command docker kill <container-id>.
If you type in docker ps after stopping the container, you should see that no containers are running.
