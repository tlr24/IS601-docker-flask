AAA Testing

AAA is a pattern of testing that helps determine whether the functions in a program are working as expected.

To start, AAA stands for Arrange, Act, and Assert.

  • Arrange
  • Act
  • Assert
  • Helpful Links

WHY TEST?

Testing is important because it helps us pinpoint problem areas in our code in development before deploying problematic code.

It also allows us to keep track of where code breaks as APIs change and new code is added later on.

It cuts down the time developers spend hunting down the issues in an application.


Ultimately, writing a test for every function in the code allows the developer to make sure that everything works as expected and is easy to fix if not.

Our calculator program includes tests for each function based on the AAA testing pattern discussed below.


For this article, we will be looking at the add() function, found in calculator/__init__.py.

STEP 1: ARRANGE

Arranging is the process of arranging the inputs and targets needed to set up for the test cases.

To test the functionality of a basic unit of code, we need to make sure we have the necessary operations, settings and/or variables set up and prepped.

For our example function from the calculator class, the arranging step should look like this:

values_tuple = (5, 2)

Here, we are arranging a tuple for the add() method as it takes in a tuple of numbers as input for the add functionality. We need to have a tuple of values to add.


STEP 2: ACT

The Act stage is where we call the function to be tested. This is the function or method call of the unit of code we are testing. The action is called so we can target whether it is behaving correctly or not in the next step.

In our example, we would be calling the add() function that we are testing using our arranged values like this:

Calculator.add(values_tuple())




STEP 3: ASSERT

Lastly, the Assert stage is where we write an assert statement to make sure we get our expected outcome as a result of our function call.

The assert function verifies that we received the correct response for the behavior. It lets us determine if the test passes or fails.


In our example, we would call the following assert statement to make sure that the last calculation result value is 7 as is expected of adding the numbers we fed the function in the arrange stage:

assert Calculator.get_last_result_value() == 7


If we run this test (using pylint) we should be able to see if our function passes the test and works as we expect it to.

This helps us narrow down issues in our code and makes sure that every unit of code in our program functions properly.


Each function in a program should have a test function starting with the name starting with 'test_' in order for us to run pytest and see if our code works as expected.





You can learn more about pytest here: https://docs.pytest.org/en/6.2.x/contents.html.
